# Review Skill

Use Chorus review behavior to track automated reviewer-shell comments:

1. Confirm the proposal or task being reviewed
2. Check the automated reviewer-shell comment and persisted review state
3. Keep review findings separate from implementation notes when comments include actionable feedback
4. Address accepted findings with focused changes and verification

The plugin posts placeholder PASS reviewer-shell comments, persists review verdict state, and routes notifications; do not emulate those internals manually.
