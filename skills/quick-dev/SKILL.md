# Quick Development Skill

Use quick development for narrow, low-risk changes:

1. Confirm the change is small and already approved or clearly requested
2. Make the minimal correct edit
3. Run focused verification
4. Summarize the change and any remaining risk

Escalate to the proposal workflow when scope, dependencies, or acceptance criteria are unclear.
